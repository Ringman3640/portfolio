Description

The CARLA Platoon System is an implementation of automated longitudinal control within a vehicle platoon. It was developed within the CARLA simulator and programming with Python using CARLA's Python API.

This project was started as an individual assignment for an automated vehicles course at university. The total development time, including research and implementation, was approximately one month.

How It Works

In a vehicle platoon, there are two types of vehicels: the lead vehicle and following vehicles. The lead vehicle is the vehicle at the front of the platoon formation. It is manually controlled and directs the whole platoon. Following vehicles are any vehicles behind the lead vehicle. Following vehicles are completely automated.

When the platoon is spawned into the CARLA simulated world, the lead vehicle is given several path options. Based on the selected path, the lead vehicle will start accelerating while broadcasting its movements to the following vehicles using a simple Python echo server.

Following vehicles will attempt to syncronize with the lead vehicle's movements. Each individual following vehicle will also act independently if it detects it is too close or too far from the vehicle in front of it. Other following vehicles will be notified of any corrective movements.

Final Results

The platoon model has shown the ability for following vehicles to react to movements from the lead vehicle and other nearby following vehicles...

...albeit a high chance of collisions.

Following vehicles have a significant chance of colliding with their front vehicle when a braking maneuver is taking place. This is exacerbated as the number of vehicles in the platoon increases. Typically, a following vehicle may speed up if they are lagging behind the platoon. However, if the platoon starts braking, then the speeding following vehicle will be too fast and cause a collision.

A marjor limitation of this platooning model is the lack of automated lateral control. Following vehicles only perform acceleration movements automatically. They do not account for steering.

Development Issues

CARLA Ease-Of-Use

I have personally be spoiled since I have only been exposed to well-documented tooling so far. CARLA was a slap to reality as the first environment I have used with very minimal, auto-generated documentation pages.

CARLA also doesn't have a lot of convenience methods in its API. This made it impossible to fine-tune the automated vehicle controls since I had no knowledge or experience with PID controllers at the time.

CARLA Build Time

It takes several hours to setup and build the CARLA simulator. This is because the simulation and other necessary resources are multiple gigabytes in size. The setup also requires building a custom version of the Unreal Engine.

The long build time made me hesitate to mess with modifying anything in the CARLA world. I didn't want to risk breaking something and needing to rebuild the project. This ultimately prevented me from creating a custom testing map.

Unreal Engine Complexity

I expected Unreal Engine to be relatively simple to understand coming from Unity. However, there were significant differences that prevented me from being able to easily add a custom CARLA sensor to the simulation. I ultimately had to drop the sensor idea for time reasons.

First Time With Python

This project was my first time using Python. I spend a lot of time going through the documentation of libraries. My code organization was also very cluttered.

Focus On Other Project

At the time I was working on the platoon system, I was also working on another project (hint: you're looking at it). I prioritized that other project, leaving me with less time to research and develop the platoon system.
